<table><tr><td> <table> <tr> <td colspan="2"> <p>По каналу связи передаётся последовательность целых чисел – показания прибора. В течение <i>N</i> мин. (<i>N</i> – натуральное число) прибор ежеминутно регистрирует значение напряжения (в условных единицах) в электрической сети и передаёт его на сервер. </p> <p>Определите три таких переданных числа, чтобы между моментами передачи любых двух из них прошло <b>не менее</b> <i>K</i> мин., а сумма этих трёх чисел была максимально возможной. Запишите в ответе найденную сумму.</p> <p><i>Входные данные</i></p> <p>Даны два входных файла (файл <i>A</i> и файл <i>B</i>), каждый из которых 
в первой строке содержит натуральное число <i>K – </i>минимальноеколичество минут, которое должно пройти между моментами передачи показаний, а во второй – количество переданных показаний <i>N</i> (1 ≤ <i>N</i> ≤ 10 000 000, <i>N </i>&gt;<i>K</i>). В каждой из следующих
<i>N</i> строк находится одно целое число, по модулю не превышающее 10 000 000, которое обозначает значение напряжения в соответствующую минуту.</p> <p>Запишите в ответе два числа: сначала значение искомой величины для файла <i>А</i>, затем – для файла <i>B</i>.</p> <p></p> <p><i>Типовой пример организации данных во входном файле</i></p> <p><i>2</i></p> <p><i>6</i></p> <p><i>150</i></p> <p><i>–150</i></p> <p><i>20</i></p> <p><i>–200</i></p> <p><i>–300</i></p> <p><i>0</i></p> <p><i>При таких исходных данных искомая величина равна 170 – это сумма значений, зафиксированных на первой, третьей и шестой минутах измерений.</i></p> <p><b>Типовой пример имеет иллюстративный характер.Для выполнения задания используйте данные из прилагаемых файлов.</b></p> <p><b>Предупреждение:</b> для обработки файла <i>B</i> <b>не следует</b> использовать переборный алгоритм, вычисляющий сумму для всех возможных вариантов, поскольку написанная по такому алгоритму программа будет выполняться слишком долго.</p> </td> </tr> </table> </td></tr></table>
